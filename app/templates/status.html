Troubleshooting Guide

AI Analysis Not Working

Using Ollama:

1. Ensure Ollama is running: ollama serve

2. Pull a model: ollama pull llama3.2

3. Check the endpoint matches your .env: OLLAMA_BASE_URL

4. If analysis is slow, try a smaller model or increase OLLAMA_TIMEOUT

Using LM Studio:

1. Open LM Studio and start the Local Server (Developer tab)

2. Load a model in LM Studio before starting the server

3. Check endpoint matches your .env: LMSTUDIO_BASE_URL (default: http://127.0.0.1:1234/v1)

4. Set LLM_PROVIDER=lmstudio in .env to use it as the active provider

5. Verify LMSTUDIO_MODEL matches the loaded model ID

6. Restart the app with --lmstudio flag after changing settings

Voice Transcription Not Working

1. Install Whisper: pip install openai-whisper

2. Install ffmpeg (required for audio processing)

3. Grant microphone permissions in your browser

4. For better accuracy, use a larger model in Settings

Semantic Search Not Finding Results

1. Ensure ChromaDB is operational (check above)

2. Entries need to be indexed - older entries may need re-indexing

3. Try keyword search as a fallback

4. If ChromaDB is corrupted, delete the folder and restart

Database Errors or Corruption

1. Export your data first (Settings > Export)

2. Try running: sqlite3 journal.db "PRAGMA integrity_check"

3. If corrupted, restore from backup or delete and re-import

4. Enable WAL mode for better durability (default in this app)
